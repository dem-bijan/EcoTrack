=============================================================
           ECOTRACK: PROJECT ARCHITECTURE & PROGRESS REPORT
=============================================================
Last Updated: 2026-05-08

1. EXECUTIVE SUMMARY
EcoTrack is a state-of-the-art sustainability and carbon footprint tracking platform. It leverages an AI-driven conversational interface (the "Sustainability Auditor") to seamlessly extract ecological data from natural user input. The application features a high-performance Spring Boot backend with dual-database persistence, feeding real-time updates to a premium, glassmorphic Next.js frontend via WebSockets.

2. TECHNOLOGY STACK & ARCHITECTURE
- Frontend: Next.js (App Router), React 19, Tailwind CSS, Framer Motion, Recharts, Lucide Icons.
- Backend: Java 17+, Spring Boot, Spring Security, Spring WebSockets (STOMP).
- Primary Database (SQL): PostgreSQL (handles relational business logic, users, logs, and footprints).
- Ephemeral Database (NoSQL): MongoDB (handles fast, document-based 24-hour chat history persistence).
- AI Engine: NVIDIA Integrate API (GPT-OSS-120B) for zero-latency NLP data extraction.

3. COMPLETED MILESTONES & CORE FEATURES

[A] UI/UX & Design System
- Engineered a cohesive, premium "Glassmorphism" aesthetic using semi-transparent surfaces, dynamic backdrop blurs, and an immersive dark mode palette (Slate/Emerald/Blue).
- Implemented Framer Motion for buttery-smooth page transitions and micro-animations.
- Built custom responsive layouts: Auth forms, Onboarding wizards, and a centralized Dashboard command center.
- Designed custom sleek scrollbars and sticky headers for dense data components to preserve vertical screen real estate.

[B] AI "Sustainability Auditor" Integration
- Transitioned from local Ollama instances to NVIDIA's cloud API for reliable, lightning-fast inference.
- Engineered a strict System Prompt enforcing a specific AI persona.
- Developed a highly customized regex and JSON parser within `AiController` to invisibly extract formatted strings (`[LOG: {"co2": ...}]`, `[FP: ...]`, `[REC: ...]`) from conversational text.
- Ensured AI responses dynamically calculate baseline footprints and activity deltas using standardized ecological formulas.

[C] Authentication & Security
- Built a robust JWT-based authentication pipeline.
- Backend: Configured `JwtAuthenticationFilter` to secure endpoints, extract tokens, and populate Spring's `SecurityContextHolder`.
- Frontend: Next.js middleware/server-side components securely extract JWTs from HTTP-only cookies and protect routing logic.

[D] Data Persistence & Analytics
- Chat History: Integrated MongoDB via `SessionService` to map conversation logs to users, auto-pruning messages older than 24 hours to save space.
- Activity Logs: Successfully map AI `[LOG:]` outputs directly into the PostgreSQL `activities` table.
- Carbon Footprint Engine: Calculates a daily *average* carbon footprint (averaging multiple updates per day via Java Streams) to plot a precise, smooth historical trajectory without duplicate data points.

[E] Real-Time Event Driven Architecture
- Implemented Spring Boot WebSockets with STOMP broadcasting.
- When the AI logs a new activity or updates the footprint, the backend fires a `REFETCH` ping to the specific user's private `/topic/updates/{email}` channel.
- The React frontend uses `sockjs-client` and `@stomp/stompjs` to listen for these pings, triggering instant data re-fetches.
- Results in real-time `TrendChart` and `LiveActivityTable` updates with zero page-reloads.

4. CURRENT DASHBOARD COMPONENTS
- Carbon Score Gauge: Displays total live cumulative footprint in Tons.
- TrendChart: A customized Recharts AreaChart with SVG glow filters, mapping daily footprint averages.
- LiveActivityTable: A vertically scrollable, sticky-header table showing all logged events, classifications, timestamps, and kg deltas.
- Metrics Ribbon: Quick stats highlighting total footprint, efficiency index vs baseline, and active log counts.
